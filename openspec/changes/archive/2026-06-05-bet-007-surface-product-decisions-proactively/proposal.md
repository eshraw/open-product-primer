## Context

- Bet: BET-007
- Bet decision: oprim/bets/BET-007/bet-decision.md

## Why

Users working on bets, discovery, or specs have no way to know which existing PDRs are relevant to their current work — decisions are siloed in `oprim/decisions/` and only surface if the user remembers to look. Because the agent has full conversation context, it can match that context against PDR titles and content to surface relevant decisions proactively at the moment they matter.

## What Changes

- `oprim init` and `oprim update` gain an opt-in prompt: "Enable proactive PDR surfacing in skills? (y/N)" — disabled by default
- When opted in: a new `oprim:context` skill is installed that scans `oprim/decisions/`, extracts topic keywords from the current conversation, and surfaces any PDRs whose titles or content overlap with those keywords
- When opted in: all oprim skills (`oprim-bet`, `oprim-pdr`, `oprim-review`, `oprim-criteria`, `oprim-archive`) are installed with a first step that invokes `oprim:context`
- When opted in: all openspec skills (`openspec-propose`, `openspec-apply-change`, `openspec-explore`, `openspec-archive-change`) are installed with a first step that invokes `oprim:context`
- When opted out (default): skills are installed exactly as today — no `oprim:context` invocation, no behavior change
- If `oprim/decisions/` is empty at runtime, the skill exits silently without prompting the user

## Capabilities

### New Capabilities

- `pdr-proactive-surfacing`: An opt-in feature gated at `oprim init`/`update` time — installs a new `oprim:context` skill that reads `oprim/decisions/`, uses keyword matching against the active conversation, and surfaces relevant PDRs as a compact inline list before the invoked skill executes. Disabled by default.

### Modified Capabilities

- `pdr-authoring`: The `oprim-pdr` skill SHALL invoke `oprim:context` at the start of execution so existing decisions are visible before authoring a new one

## Impact

- `packages/cli/src/lib/install-agent.ts` — all oprim and openspec skill string constants get a new "Check relevant PDRs" step; the new `oprim:context` skill string constant is added here
- `.claude/skills/oprim-*/SKILL.md` (regenerated by `oprim update`) — each skill gains a first step invoking `oprim:context`
- `.claude/skills/openspec-*/SKILL.md` (regenerated by `oprim update`) — each skill gains a first step invoking `oprim:context`
