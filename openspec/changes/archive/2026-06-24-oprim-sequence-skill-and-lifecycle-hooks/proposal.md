## Why

The sequencing board falls out of date because editing `sequence.yaml` by hand is mechanical, there is no trigger that fires when something changes (a bet archived, a bet created), and `/oprim:sequence` requires users to arrive with a clear intention — but users often don't know what action to take. The result: the board drifts, and nobody sequences.

## What Changes

- **New `oprim-sequence` skill** — replaces the inline steps in the current `sequence.md` command. Owns board health computation, triage mode (no intention), seeded mode (context from hook), and YAML execution with diff preview.
- **Refactored `sequence.md` command** — becomes a one-line wrapper invoking the `oprim-sequence` skill, consistent with `archive.md`.
- **Lifecycle hooks extended** — `on-prompt-submit.sh` detects `/oprim:bet` and `/oprim:promote` invocations and writes `.sequence-nudge` flag files with context. `on-stop.sh` reads flags and surfaces non-blocking nudges that pre-seed the skill with what just happened.
- **`install-agent.ts` updated** — source of truth for all three surfaces above.

## Capabilities

### New Capabilities

- `oprim-sequence-skill`: The `oprim-sequence` skill behavior — triage mode, seeded mode, board health computation, move validation, and YAML execution.
- `sequencing-lifecycle-hooks`: Hook contracts for detecting bet lifecycle events and surfacing non-blocking sequencing nudges at session stop.

### Modified Capabilities

- `sequencing-board`: The interaction model for updating the board changes — from a user-driven command with inline steps to a skill-driven flow with proactive entry points.

## Impact

- `packages/cli/src/lib/install-agent.ts` — skill string for `oprim-sequence`, updated command wrapper, updated hook scripts
- `.claude/skills/oprim-sequence/SKILL.md` — new file (generated by `oprim update`)
- `.claude/commands/oprim/sequence.md` — simplified to skill wrapper (generated by `oprim update`)
- `.claude/hooks/on-prompt-submit.sh` — new detections for `/oprim:bet` and `/oprim:promote`
- `.claude/hooks/on-stop.sh` — new flag-read and nudge output for `.sequence-nudge`
